- 그날의 작업 내용을 `계획 / 결정 / 실행 / 질문 / 검증 / 다음 작업` 기준으로 한 번에 훑을 수 있게 정리한다.

## 오늘 결정한 것

- Codex 일일 기록은 `docs/journal/codex/YYYY-MM-DD.md` 형식으로 남긴다.
- 요약 기준은 `오늘 계획`, `오늘 결정한 것`, `오늘 실행한 것`, `오늘 Codex에 물어본 것`, `검증`, `다음에 이어서 할 것`으로 둔다.
- `docs/legacy/phase-harness/phases/**`는 실행 대상이 아니라 아카이브로 유지한다.
- 현재 실행 기준은 `missions/active/*.md`와 `missions/completed/*.md`다.
- Mission 001은 더 이상 구조만 있는 skeleton이 아니라, router / planner / executor / approval / evidence / final response / eval까지 검증되는 완료 후보로 본다.
- Mission 001은 `missions/completed/001-agent-runtime-skeleton.md`로 이동해 완료 처리한다.
- Mission 001 이후 바로 진행할 후속 작업은 두 개로 잡는다.
- `006-workflow-audit-hardening.md`: idempotency, approval 우회 방지, Evidence Log 순서/필수 이벤트, PII 로그 방지 강화.
- `007-frontend-api-integration.md`: Mission 005 dashboard skeleton을 backend API와 연결.
- `synthetic-case-fixtures`와 `rag-retrieval-upgrade`는 후속 후보로 남기되, 지금 바로 active mission으로 올리지는 않는다.
- `FOLDER_STRUCTURE.md`는 구조 변경이 있을 때 함께 갱신한다.
- 빈 폴더의 `.gitkeep`은 유지하되, 실제 파일이 생긴 폴더의 `.gitkeep`은 제거해도 된다.
