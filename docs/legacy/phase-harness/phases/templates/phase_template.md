# 단계 번호: 제목

## 목표
이 Phase에서 달성할 가장 작은 사용자 가시 결과 또는 harness 가시 결과를 적는다.

## 입력
- `AGENTS.md`
- 이 Phase가 의존하는 프로젝트 문서 또는 데이터 파일

## 허용 쓰기
- 이 Phase가 만들거나 수정할 수 있는 정확한 파일 또는 디렉터리를 적는다.

## 금지
- 변경해서는 안 되는 파일, 디렉터리, 명령, 의존성, 동작을 적는다.

## 단계
1. 필요한 입력을 확인한다.
2. 범위 안의 변경만 구현한다.
3. 검증을 실행한다.
4. 상태와 위험을 보고한다.

## 검증
다음을 실행한다:

```bash
uv run pytest
```

## 중단 조건
- 필요한 입력이 없다.
- 필요한 쓰기 경로가 `Allowed Writes`에 없다.
- 검증을 실행할 수 없거나, 이 Phase 밖의 이유로 실패한다.

## 최종 출력 요구사항
응답의 마지막은 다음 중 하나여야 한다:

```text
status: completed
status: blocked
status: error
```
