# 1C단계: Chunking + Dense Retrieval

## 목표
token-overlap baseline retriever를 structured chunk와 dense retrieval pipeline으로 바꾼다. Phase 1은 source-level baseline에서 `Hit@3 = 83%`를 달성했지만, chunking과 `case_type` metadata가 없어서 Phase 3/4의 evidence citation 품질이 제한된다.

## 입력
- `01_workforce_agent_schema.md` (chunk_type rules and evidence_grade)
- `docs/DATA_GUIDE.md` (regulation_chunks.jsonl schema and sources.json schema)
- `data/raw/source_manifest.json` (5 HTML + 1 PDF official sources, frozen)
- `data/processed/sources.json` (existing, update case_type metadata)
- `eval/retrieval_eval.jsonl` (12 questions, expected source_id + case_type)
- `eval/results/phase1_retrieval_report.md` (baseline = 83%)

## 금지
- 새 official source를 추가하지 않는다. source manifest는 Phase 1에서 고정된다.
- LLM을 호출하지 않는다. Embedding은 LLM 호출이 아니며 OpenAI `text-embedding-3-small`만 사용한다.
- synthetic case(`data/synthetic_cases/`)를 공식 chunk index에 포함하지 않는다.
- chunk text를 지어내지 않는다. 모든 chunk는 `data/raw/`에서 가져와야 한다.
- `DATA_GUIDE.md` schema를 변경하지 않는다. 기존 계약에 맞게 구현한다.
- 다음 explicit dependency 목록 밖의 패키지를 추가하지 않는다: `openai`, `chromadb`, `rank-bm25`, `pdfplumber`, `python-dotenv`.
- `pyproject.toml`에서 프로젝트 이름, 설명, 기타 무관한 metadata를 수정하지 않는다.

## 단계

### 1. Chunking (`src/chunk.py`)
`source_manifest.json`의 각 source에 대해 `data/raw/`에서 텍스트를 파싱하고 다음 기준으로 나눈다:

- `law_*`: clause-level chunks with `chunk_type: "law_clause"`.
- `eps_employer_process_*` and `eps_application_guide_*`: procedure-step chunks with `chunk_type: "procedure_step"`.
- `eps_allowed_industries_*`: industry-entry chunks with `chunk_type: "industry_entry"`.
- `law_form_*`: form-field chunks with `chunk_type: "form_field"`.
- `eps_employer_scoring_*`: scoring-criterion chunks with `chunk_type: "scoring_criterion"`.

각 chunk record에는 `chunk_id`, `source_id`, `chunk_text`, `chunk_type`, `section_title`, `order_in_doc`, `char_count`가 포함되어야 한다.

`data/processed/regulation_chunks.jsonl`에 한 줄당 JSON object 하나씩 기록한다.

### 2. Sources Metadata Correction
현재 `sources.json`는 모든 source에 `case_type: ["ALL"]`이 들어 있어 case_type filtering이 작동하지 않는다. source별로 다음처럼 수정한다:

- `eps_employer_process_001` -> `["new_hiring", "rehire_loyalty"]`
- `eps_allowed_industries_001` -> `["ALL"]`
- `eps_application_guide_001` -> `["new_hiring"]`
- `eps_employer_scoring_001` -> `["new_hiring", "workplace_change_intake"]`
- `law_foreign_worker_act_001` -> `["ALL"]`
- `law_form_employment_change_001` -> `["workplace_change_intake", "rehire_loyalty"]`

### 3. Embedding (`src/embed.py`)
- `python-dotenv`의 `load_dotenv()`를 호출한 뒤 `os.environ`에서 `OPENAI_API_KEY`를 읽는다.
- OpenAI `text-embedding-3-small`를 사용한다.
- 각 chunk의 `chunk_text`를 embedding한다.
- `data/index/`의 local Chroma collection에 저장한다.
- 각 vector record에 chunk와 source metadata를 붙인다.

### 3a. Embedding Cache (`src/cache/embedding_cache.py`)
- cache key로 `sha256(chunk_text)`를 사용한다.
- cache hit이면 `data/cache/embeddings.sqlite`에서 embedding을 불러온다.
- cache miss이면 OpenAI를 호출한 뒤 embedding을 sqlite에 저장한다.

### 4. Hybrid Retriever (`src/retrieve.py` refactor)
- `rank_bm25`를 사용해 chunk text 위에서 BM25 retriever를 만든다.
- Chroma 위에서 dense retriever를 만든다.
- weighted score로 합친다 (0.4 BM25 / 0.6 dense).
- 지원 filter: `visa_type`, `case_type`, `evidence_grade`.
- 시그니처: `search(query, k=5, filters={}) -> list[ChunkResult]`.
- 각 결과는 `chunk_id`와 `source_id`를 노출해야 한다.

### 5. Re-evaluate
- 새 retriever에 대해 `evaluate.py`를 실행한다.
- `eval/results/phase1c_retrieval_report.md`를 쓴다.

## 검증
다음을 모두 실행한다:

```bash
uv run pytest tests/test_chunk.py tests/test_retrieve.py
uv run python -m src.chunk
uv run python -m src.embed
uv run python -m src.evaluate
```

Required:
- `data/processed/regulation_chunks.jsonl`이 존재하고 chunk가 총 30개 이상이어야 한다.
- 모든 chunk의 `chunk_type`이 DATA_GUIDE.md 허용 목록에 있어야 한다.
- `data/index/`가 존재하고 non-empty Chroma collection이 있어야 한다.
- `eval/results/phase1c_retrieval_report.md`에 `Hit@3 >= 90%`가 보고되어야 한다.

## 중단 조건
- DATA_GUIDE.md schema에 없는 `chunk_type` 값이 생성되면 blocked로 표시한다.
- 빈 text이거나 `source_id`가 없는 chunk가 생성되면 blocked로 표시한다.
- baseline 대비 `Hit@3`가 80% 아래로 떨어지면 blocked로 표시한다.
- synthetic case가 실수로 공식 collection에 인덱싱되면 error로 표시한다.
- OpenAI embeddings 외에 새 web fetch 등 다른 network access가 필요하면 blocked로 표시한다.
- missing metadata에 대한 silent fallback은 금지한다. 명시적 이유와 함께 blocked로 표시한다.
